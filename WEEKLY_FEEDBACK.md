# 週次トレンドフィードバック自動配信システム（提案 & セットアップ）

既存の「日次X競合分析（無料版）」を**ベース**に、**毎週月曜の朝、競合の1週間の動きを総括した
レポートをメールで自動受信する**仕組みを追加しました。月額コストは引き続き **¥0** です。

---

## 1. 何ができるようになるか

```
毎週月曜 7:30 JST
  └─ GitHub Actions が自動起動（あなたは何もしない）
      1. 直近7日分の日次レポート(reports/)を自動収集
      2. Gemini が「1週間を貫くトレンド・変化・打ち手」を総括
      3. あなたのメールに1通だけ届く
           ┌─────────────────────────────┐
           │ 件名: 📊 競合X週次トレンド総括（6/09〜6/15） │
           │                                       │
           │ 🧭 今週の要点（3行）                      │
           │ 🔥 今週バズった型 TOP3                    │
           │ 📈 共通トレンド & 先週からの変化            │
           │ 🗺 ポジショニング所感（空いている余地）       │
           │ 🎯 今週の戦略フィードバック（3点）           │
           └─────────────────────────────┘
  └─ 月曜の朝、メール1通で「先週の競合」と「今週の方針」が掴める
```

- **配信先**：メール（Gmail / SMTP）
- **内容の重点**：競合の週次トレンド総括（依頼者の差別化軸＝経営者×実務深掘りに沿った戦略提言つき）
- **日次（Telegram）はそのまま稼働**。週次はそれを束ねる“上位レイヤー”として併走します。

---

## 2. なぜこの設計か（提案の根拠）

| 論点 | 採用した方針 | 理由 |
|------|------------|------|
| データ源 | 日次レポート(reports/)を再利用、無ければRSSにフォールバック | 既に毎日溜まる成果物を二次利用＝追加コスト・追加API呼び出しゼロ。日次が止まっていても週次は単独で動く |
| 実行基盤 | 既存と同じ GitHub Actions（cron） | 新しいインフラ不要。日次と同じ運用・同じSecretsでOK |
| AI | 既存と同じ Gemini 2.5 Flash | 無料枠内（1日250リクエストに対し週1回）。追加費用なし |
| 配信 | メール（SMTP / stdlibのみ） | ご希望の経路。長文・表・リンクが読みやすく、検索・保存に強い。新規ライブラリ追加なし |
| 実行時刻 | 月曜 7:30 JST（日次の7:00の30分後） | その日の日次レポートがcommitされた後に走らせ、最新7日分を確実に取り込むため |

## 3. セットアップ（日次が動いている前提なら約5分）

日次システムで **`GEMINI_API_KEY`** は設定済みのはずなので、追加で必要なのは**メール送信用のSecretsだけ**です。

### 3-1. Gmailのアプリパスワードを取得
1. Googleアカウントで「2段階認証」を有効化（未設定の場合）
2. https://myaccount.google.com/apppasswords で「アプリパスワード」を発行（16桁）
   - ※通常のログインパスワードは使えません。必ずアプリパスワードを使用します。

### 3-2. GitHub Secrets を追加
リポジトリの **Settings → Secrets and variables → Actions → New repository secret** で以下を登録：

| Secret名 | 値の例 | 必須 |
|---------|-------|------|
| `SMTP_USER` | `yourname@gmail.com`（送信元Gmail） | ✅ |
| `SMTP_PASS` | 上記で発行した16桁アプリパスワード | ✅ |
| `MAIL_TO` | `ys.saitoh2432@gmail.com`（受信先。カンマ区切りで複数可） | ✅ |
| `SMTP_HOST` | `smtp.gmail.com`（未設定ならこの値が既定） | 任意 |
| `SMTP_PORT` | `465`（未設定なら465。STARTTLSなら`587`） | 任意 |
| `MAIL_FROM` | 送信元アドレス（未設定なら`SMTP_USER`） | 任意 |
| `MAIL_FROM_NAME` | `X競合分析Bot`（差出人表示名） | 任意 |

> Gmail以外（独自ドメイン等）でも `SMTP_HOST` / `SMTP_PORT` を変えれば利用できます。

### 3-3. 動作テスト
**Actions タブ → 「Weekly X Competitor Feedback」→ Run workflow** で手動実行。
数十秒後にメールが届けば成功です（同時に `weekly_reports/YYYY-MM-DD.md` がリポジトリに保存されます）。

## 4. カスタマイズ早見表

| 変えたいこと | 編集する場所 |
|------------|-------------|
| 配信曜日・時刻 | `.github/workflows/weekly-feedback.yml` の `cron`（既定：日曜22:30 UTC=月曜7:30 JST） |
| 集約する期間 | `scripts/weekly_feedback.py` の `LOOKBACK_DAYS`（既定7日） |
| レポートの観点・章立て | `prompts/weekly_system_prompt.txt` |
| 受信先・差出人名 | GitHub Secrets（`MAIL_TO` / `MAIL_FROM_NAME`） |
| 監視する競合 | `config/competitors.json`（日次と共通） |

## 5. 運用上の注意

- **メールが届かない場合**：①Secrets名の打ち間違い、②2段階認証＋アプリパスワード未使用、③`MAIL_TO`未設定、の順に確認。失敗時は同じSMTP設定で「失敗通知メール」が飛ぶよう実装済み。
- **週次が空っぽになる場合**：日次レポートが7日間1件も無いとRSSフォールバックに切り替わります。RSSHub公開インスタンスが不安定だと内容が薄くなることがあります（その旨レポートの「📝 メモ」に明記されます）。
- **倫理面**：競合投稿の丸写しはしません。「型・構造を借りて自分の体験で書き直す」前提でプロンプトを設計しています。

# my_posts/ — 自分の過去X投稿（過去記事情報）置き場

このフォルダに入れた過去投稿は、毎回の自動実行時にAIへ「過去記事情報」として渡されます。
用途は **重複ネタの回避** と **過去投稿の再活用提案** です。

## 入れ方

ローカルで管理している過去投稿を、このフォルダにコピーして push するだけです。

```bash
git add my_posts/
git commit -m "add past posts"
git push
```

## 対応フォーマット

`README.md` と `.` で始まるファイルは無視されます。それ以外のファイルを次の形式で置けます。

### 1. テキスト（`.txt` / `.md`）
- 1ファイル = 1投稿が基本。
- 1ファイルに複数投稿を入れたい場合は、**`---` だけの行**で区切ってください。

```
生成AIを導入した中小企業の現場で実際に起きたこと。最初の1ヶ月は…（本文）
---
コンサル11年でわかった、経営者がAI活用でつまずく3つのポイント。…（本文）
```

### 2. JSON（`.json`）
次のいずれの形でもOK。

```json
["1つ目の投稿本文", "2つ目の投稿本文"]
```
```json
{ "posts": [ { "text": "投稿本文", "posted_at": "2026-05-01" } ] }
```

### 3. CSV（`.csv`）
`text`（または `tweet` / `本文`）の列を含めてください。`posted_at` / `date` / `日付` 列は任意。

```csv
text,posted_at
"投稿本文1",2026-05-01
"投稿本文2",2026-05-03
```

## 上限の調整
`config/competitors.json` の `settings` で調整できます。
- `max_my_posts`：読み込む過去投稿の最大件数（デフォルト200）
- `max_my_post_chars`：1投稿あたりの最大文字数（デフォルト500）

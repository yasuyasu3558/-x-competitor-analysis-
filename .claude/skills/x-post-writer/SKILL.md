# x-post-writer — 毎日のX投稿を過去データ参考で作るスキル

ユーザーが「今日のX投稿を作って／ツイート案を出して／投稿を練りたい」等を依頼したら、このスキルを使う。

## 手順

1. **参考データ（過去のX投稿）を読み込む**
   - まず `.claude/skills/x-post-writer/reference_digest.md` を読む。これは `x_reference/`（＝ローカルの「Xについて」フォルダの同期先）から **1日1回** 自動生成されるダイジェスト。
   - ダイジェストが無い／古い場合は `x_reference/` 内のファイルを直接読んで補う。
   - 注意：ダイジェストの「最終更新」が今日でなくても、それはスキルの仕様（1日1回更新）。勝手に再生成しない。再生成が必要なら `python scripts/build_skill_digest.py --force` を案内する。

2. **発信者プロフィールを踏まえる**（`config/competitors.json` の `user_profile`）
   - 立場：代表取締役・マーケティングコンサル兼 生成AI活用アドバイザー（マーケコンサル11年）
   - 差別化：経営者×実務深掘り（B2B・中小企業向け）
   - ターゲット：中小企業経営者・個人事業主・副業ガチ勢（B2B寄り）
   - トーン：一人称・体験談ベース・経営者目線
   - ハッシュタグ：#起業 #生成AI #マーケティング #経営者

3. **投稿案を作る（ルール厳守）**
   - **重複回避**：参考データの過去投稿とテーマ・切り口・具体例が被らないこと。近い場合は角度を変える／続編にする／より深掘りする。
   - **文体の再現**：過去投稿の一人称・語り口・癖に合わせ「本人が書いたように」自然に。
   - 各案は **280字以内（全角換算）**・そのままコピペ可・ハッシュタグ込み。
   - 既定では **3案** 提示し、最後に「一押し1案＋理由」を添える。

4. **過去投稿の再活用提案**
   - 参考データの中から、いま再投稿・リライト・続編にすると効きそうなものを1つ選び、「元ネタ要約 → 今日の角度でのリライト案（280字以内）」を提案する。

## 出力フォーマット（Markdown）

```
## ✍️ 今日のX投稿案
【案A】… (280字以内・ハッシュタグ込み)
【案B】…
【案C】…

## 🎯 一押し
案X：理由（1〜2文）
本文（コピペ可で再掲）

## ♻️ 過去投稿の再活用
元ネタ：…（要約）
リライト案：…（280字以内）
```

## やってはいけないこと
- 競合や他人の投稿をそのまま転載しない。「構造を借りて自分の体験で書き直す」。
- 過去投稿と内容が重複した案を出さない。
- 280字を超える案を出さない。

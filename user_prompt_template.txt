# X競合分析 完全自動化システム（無料版）

副業ジャンルのX競合4アカウントを毎朝7:00 JSTに自動分析し、Telegramに配信。**月額¥0**。

---

## このパッケージの中身

```
competitor-x-analysis-free/
├── README.md                          ← このファイル
├── SETUP.md                           ← 15分セットアップ手順
├── requirements.txt                   ← Python依存パッケージ
├── .gitignore                         ← Git管理から除外する設定
├── .github/
│   └── workflows/
│       └── daily-analysis.yml         ← GitHub Actions定義(毎朝7時実行)
├── scripts/
│   └── fetch_and_analyze.py           ← メインスクリプト(RSS→Gemini→配信)
├── prompts/
│   ├── system_prompt.txt              ← Geminiのシステムプロンプト
│   └── user_prompt_template.txt       ← ユーザープロンプト雛形
├── config/
│   └── competitors.json               ← 4アカウントの設定
└── reports/                           ← 毎日の分析結果が自動で蓄積
    └── (空フォルダ。実行後にMDファイルが追加される)
```

## よくある質問

### Q. GitHubリポジトリをPublicにすると恥ずかしい

**A.** 公開されるのは「コード」と「分析結果のMD」のみ。APIキーやアカウント情報はGitHub Secretsに暗号化保存されるので外部からは見えない。
気になる場合はPrivateリポジトリでも動くが、月2,000分のActions実行制限内（本システムは月60分なので余裕）。

### Q. RSSHub公開インスタンスが落ちたら？

**A.** スクリプトは複数のミラーURLを順に試すフォールバック実装あり。それでも全滅した場合は、その日のレポートに「データ取得失敗・過去データで生成」と明記される。次の日には自動復旧することが多い。

### Q. Geminiの出力品質ってどのくらい？

**A.** Claude Sonnet 4.6比で約85〜90%。日本語の自然さは同等、戦略的洞察は若干劣る。日次の競合監視レベルなら十分実用範囲。
もし不満なら、`scripts/fetch_and_analyze.py`の中でGemini部分をClaude API呼び出しに差し替えるだけで品質UP（その場合は月¥200程度発生）。

### Q. Telegramは使ったことないけど大丈夫？

**A.** 5分で慣れる。スマホアプリ（無料）をDLし、BotFatherで自分専用のbotを作るだけ。LINEのほうが慣れている場合は、SETUP.mdのオプショナル設定でLINE通知も追加可能。

### Q. 投稿は手動でコピペが必須？

**A.** はい。Xへの自動投稿はAPI課金が必須（月¥31,000〜）になるため、無料版では「コピペで1分」の運用にしている。
朝にAIが分析→人がコピペ判断、という分業がトータルで最も効率良い。

## 法的・倫理的な注意

- RSSHubは各国の規約遵守でグレーゾーン。商用利用やスクレイピング過多は避ける
- Geminiの利用規約上、医療・法律など専門領域での重要決定には使わない
- 競合アカウントの投稿を**そのまま転載**することは絶対NG。「構造を借りて自分の体験で書き直す」が原則
